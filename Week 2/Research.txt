Models like BERT or LLaMA are very large models. How can we
address this issue?

Solutions :

Knowledge distillation — train a smaller "student" model to mimic a larger "teacher" (e.g., DistilBERT is 40% smaller than BERT, retains ~97% performance)
Pruning — remove redundant weights/neurons/attention heads
Quantization — reduce numerical precision of weights/activations
Low-rank adaptation (LoRA) — freeze base weights, train small low-rank adapter matrices (more about fine-tuning efficiency than inference size, but worth a mention)

Quantization in depth
The idea: represent weights using fewer bits (int8, int4) instead of fp32/fp16, with a scale factor mapping between ranges.



Code examples
1. Dynamic quantization on BERT with PyTorch (CPU, INT8):
pythonimport torch
from transformers import BertModel, BertTokenizer

model = BertModel.from_pretrained("bert-base-uncased")
model.eval()

quantized_model = torch.quantization.quantize_dynamic(
    model, {torch.nn.Linear}, dtype=torch.qint8
)

def get_model_size(m):
    torch.save(m.state_dict(), "tmp.pt")
    import os
    size = os.path.getsize("tmp.pt") / 1e6
    os.remove("tmp.pt")
    return size

print(f"Original size: {get_model_size(model):.1f} MB")
print(f"Quantized size: {get_model_size(quantized_model):.1f} MB")
2. Loading LLaMA in 4-bit with bitsandbytes:
pythonfrom transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
import torch

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True,
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    quantization_config=bnb_config,
    device_map="auto",
)
This drops LLaMA-7B from ~28GB (fp32) to roughly 4–5GB, fitting on a single consumer GPU.
